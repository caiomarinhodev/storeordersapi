Store Orders API

A quick way to store orders.

In this project we provide an API with a complete CRUD of Store Orders to be consumed.

In this API we will demonstrate a quick way for you to store your quotes or purchase orders data in online stores.

Here you can store all your store orders, and enjoy it for free by placing all orders generated in your store.

API

#1 Step

You will need to register to gain access to the API.

POST /sec/register

Example of payload:

{ "name": "Test User", "login": "user", "email": "user@gmail.com", "password": "123456" }

#2 Step

You will need login to gain access to the API with API Token. We use JWT to generate the token.

POST /sec/login

Example of payload:

{ "login": "user", "senha": "123456" }

You will receive a JSON response with your access token that expires in 1 hour.

{ message: 'Login realizado com sucesso', token: token, login: usuario.login, id: usuario.id, nome: usuario.nome, roles: usuario.roles }

#3 Step

We use JWT to generate our Access Tokens, which last for 1 hour. So after login you must send the access token in every request. The bearer authentication scheme was originally created as part of OAuth 2.0 in RFC 6750, but is sometimes used alone as well.

Authorization: Bearer {token}

Endpoints

Here you will have access to all orders created.

Example of response:

[ { "id": 2, "client_name": "Maria", "id_status": 1, "total_value": 10, "note": "Test order" }, { "id": 3, "client_name": "Joseph", "id_status": 1, "total_value": 10, "note": "Test order" } ]

Here you can create an order.

Example Payload to send in request:

{ "client_name": "New", "id_status": 1, "total_value": 10, "note": "New order" }

Here you can access the details of a specific order.

Here you can change the details of a specific order.

Example Paylod to send in request:

{ "client_name": "New", "id_status": 2, "total_value": 15, "note": "Update in this Order" }

Here you can remove a specific order.
